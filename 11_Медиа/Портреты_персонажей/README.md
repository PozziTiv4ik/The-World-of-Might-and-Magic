# Портреты персонажей

status: active

## Назначение

У каждого значимого персонажа должен быть основной портрет. Он нужен как визуальная опора для сцен, будущих иллюстраций и генерации новых вариантов.

## Формат портрета

- Основной портрет хранится в отдельной папке персонажа.
- Базовый визуальный стандарт: вертикальный кинематографичный портрет персонажа.
- Текущие референсы имеют размер около `1024x1536`, то есть вертикальный кадр `2:3`. По смыслу это наше персонажное "фото 3 на 4": герой крупно, лицо читается, костюм и статус видны.
- Не использовать квадратный формат для основных портретов.
- Не добавлять текст, логотипы, водяные знаки или интерфейсные элементы.

## Имена файлов

Основной портрет:

```text
11_Медиа/Портреты_персонажей/Имя_Персонажа/Имя_Персонажа_основной_портрет.jpg
```

Если нужен уточняющий статус, он добавляется в имя:

```text
Принц_Михаэль_торговый_живой_основной_портрет.jpg
```

Варианты сцен:

```text
Имя_Персонажа_сцена_краткая_тема_01.jpg
```

## Статусы в карточках

В каждой карточке персонажа должны быть поля:

```yaml
portrait: путь_к_файлу_или_null
portrait_status: available|missing|planned
```

- `available` - основной портрет есть.
- `missing` - портрета пока нет.
- `planned` - портрет нужен и уже согласован к генерации, но файла еще нет.

## Новый персонаж

Если в сюжете появляется новый значимый персонаж:

1. Создать или обновить карточку персонажа.
2. Поставить `portrait: null` и `portrait_status: missing`, если изображения еще нет.
3. Добавить персонажа в индекс портретов.
4. В ответе обязательно сказать: у нового персонажа пока нет фото.
5. Предложить сгенерировать портрет по правилам из `Правила_генерации_портретов.md`.

Короткая формулировка для ответа:

```text
У нового персонажа пока нет портрета. Могу сгенерировать его в общем стиле игры: вертикальное персонажное фото, кинематографичный реализм, с учетом роли, фракции и текущей сцены.
```

## После сюжетных сцен

После рассказа, новой сцены или важного сюжетного обновления ассистент предлагает одну визуальную идею:

```text
Визуал: сделать иллюстрацию к этой сцене? Тема: [краткая тема]. Референсы: [персонажи/портреты/локации].
```

Если ответ после обновления файлов должен завершаться меткой `СЮЖЕТ ЗАФИКСИРОВАН`, визуальное предложение ставится перед этой меткой, чтобы метка сохранения оставалась последней строкой.

## Использование референсов

- Для существующих персонажей основной портрет считается главным референсом лица, возраста, силуэта и общего настроения.
- Для сценовых иллюстраций использовать портреты участвующих персонажей как референсы идентичности.
- В новых кадрах лицо персонажа должно сохраняться максимально близко к основному портрету: важны именно черты лица, а не только костюм, цвет волос, ракурс или поза.
- Если сцена требует узнаваемого персонажа, лицо должно быть достаточно крупным и проработанным, чтобы персонажа можно было узнать без подписи.
- В сценовых иллюстрациях не добавлять случайных людей с различимыми лицами. Показывать только персонажей, указанных в сцене и референсах; для масштаба использовать дальние силуэты, спины, шлемы, знамена, корабли или архитектуру.
- В сценовых иллюстрациях персонажи не смотрят в камеру. Они смотрят друг на друга, на предметы, документы, врагов, корабли, трон, карту или другое событие внутри мира. Прямой взгляд в зрителя допустим только для отдельного основного портрета, но не для сюжетной сцены.
- Если персонаж изменился по сюжету, новый вариант сохраняется рядом с основным портретом, а не заменяет его без отдельного решения пользователя.
