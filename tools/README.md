Эта папка хранит команды, которые агент использует вместо ручной правки повторяющихся таблиц, индексов и служебных файлов.

## Режим работы агента

Пользователь может просто прислать сюжетный текст, письмо, сцену или итог решения. Агент сам выбирает нужную цепочку инструментов: принимает входящее, создает или обновляет сцену, закрывает вопросы, сдвигает фронты, пересобирает индексы и запускает проверку.

Все PowerShell-инструменты подключают `tools/_lib.ps1` и берут общую файловую блокировку проекта. Если два инструмента случайно стартуют параллельно, второй ждет завершения первого, чтобы не испортить таблицы вопросов, решений, фронтов или входящих сообщений.

## Основной цикл

- `.\tools\Принять_сообщение.ps1` - принять новый сырой текст в `07_Черновики_и_идеи/Входящие_сообщения.md`; с `-Mode source` дополнительно создает файл в `08_Источники`.
- `.\tools\Обработать_входящее.ps1` - перенести запись из новых входящих в обработанные и привязать ее к сцене, источнику или другим файлам.
- `.\tools\Сцена_из_входящего.ps1` - принять новое сообщение или взять уже лежащее во входящих, создать по нему сцену и закрыть входящую запись.
- `.\tools\Новая_сцена.ps1` - создать сцену в нужной ветке, проставить `front_id` и пересобрать индекс сцен.
- `.\tools\Завершить_ход.ps1` - пересобрать индексы, решения, вопросы, фронты, панель следующего хода и прогнать основные проверки.

## Создание сущностей

- `.\tools\Новый_персонаж.ps1` - создать карточку персонажа и добавить его в индекс персонажей.
- `.\tools\Новая_локация.ps1` - создать карточку локации и добавить ее в индекс локаций и карту регионов.
- `.\tools\Новый_портрет.ps1` - закрепить готовый портрет 3:4, обновить промпт, карточку персонажа и индекс портретов.

## Обновление кампании

- `.\tools\Закрыть_решение.ps1` - перевести `DEC-PENDING-*` в принятое решение через `09_Реестры/Решения.json`, пересобрать журнал и вопросные витрины.
- `.\tools\Собрать_решения.ps1` - пересобрать `01_Кампания/02_Журнал_решений.md` из `09_Реестры/Решения.json`; с `-ImportFromMarkdown` выполняет первичную миграцию из старого журнала.
- `.\tools\Закрыть_вопрос.ps1` - закрыть `Q-C2-*` или `Q-WORLD-*` в `09_Реестры/Вопросы.json`, добавить историю и пересобрать вопросные Markdown-витрины.
- `.\tools\Собрать_вопросы.ps1` - пересобрать `01_Кампания/03_Нерешенные_вопросы.md` и `01_Кампания/03_Закрытые_вопросы.md` из `09_Реестры/Вопросы.json`; с `-ImportFromMarkdown` выполняет первичную миграцию из старых таблиц.
- `.\tools\Новый_фронт.ps1` - создать новый `FRONT-*` в `09_Реестры/Фронты.json` и пересобрать фронтовую Markdown-витрину.
- `.\tools\Обновить_фронт.ps1` - обновить риск, состояние, следующий триггер или таймер у `FRONT-*` через JSON-реестр.
- `.\tools\Собрать_фронты.ps1` - пересобрать `01_Кампания/06_Фронты_и_таймеры.md` из `09_Реестры/Фронты.json`; с `-ImportFromMarkdown` выполняет первичную миграцию из старой таблицы.

Новые `Q-C2-*` и `Q-WORLD-*` назначай по максимуму из `09_Реестры/Вопросы.json`. Проверка проекта считает повтор ID или расхождение JSON с Markdown-витринами ошибкой.

## Индексы и проверки

- `.\tools\Собрать_индекс_сцен.ps1` - пересобрать `01_Кампания/00_Индекс_сцен.md`.
- `.\tools\Собрать_индекс_источников.ps1` - пересобрать `08_Источники/00_Индекс_источников.md`.
- `.\tools\Собрать_решения.ps1` - пересобрать журнал решений из структурного JSON-реестра.
- `.\tools\Собрать_вопросы.ps1` - пересобрать вопросные Markdown-витрины из структурного JSON-реестра.
- `.\tools\Собрать_фронты.ps1` - пересобрать фронтовую Markdown-витрину из структурного JSON-реестра.
- `.\tools\Собрать_панель_хода.ps1` - пересобрать `01_Кампания/07_Следующий_ход.md`.
- `.\tools\Проверить_архив.ps1` - проверить завершенные главы.
- `.\tools\Проверить_инструменты.ps1` - проверить ключевые инструменты на временной копии проекта, включая безопасное закрытие `DEC-PENDING-*`.
- `.\tools\Проверить_портреты.ps1` - отдельно проверить портретный слой.
- `.\tools\Проверить_проект.ps1` - общая проверка структуры, ссылок, front matter, индексов, портретов, `FRONT-ID`, устаревших активных развилок и закрытых сцен в активных списках.
- `.\tools\_lib.ps1` - общие служебные функции инструментов, включая файловую блокировку проекта.
- `.\tools\Установить_git_hooks.ps1` - подключить локальные git hooks из `.githooks`.
- `.\tools\Собрать_индекс_персонажей.ps1` - пересобрать `03_Персонажи/00_Индекс_персонажей.md` из карточек персонажей, сохраняя существующие навигационные группы.
- `.\tools\Собрать_индекс_локаций.ps1` - пересобрать `04_Локации/00_Индекс_локаций.md` из карточек локаций, используя прежний индекс как fallback для старых `front_id`.

## Примеры

```powershell
.\tools\Принять_сообщение.ps1 -Title "Новое письмо Михаэля" -TextPath ".\tmp\message.txt" -Mode source
.\tools\Новая_сцена.ps1 -Branch "Михаэль" -Title "Ответ в Крепости Сокола" -FrontId FRONT-FALCON-FORTRESS -Status active
.\tools\Сцена_из_входящего.ps1 -Branch "Александрос" -Title "Новое донесение" -TextPath ".\tmp\message.txt" -FrontId FRONT-CRIMSON-PEAK
.\tools\Новый_фронт.ps1 -Id FRONT-NEW-LINE -Name "Новая линия" -Description "кратко о фронте" -Participants "Уточнить." -State "Уточнить." -Risk "Уточнить." -Trigger "Уточнить."
.\tools\Собрать_решения.ps1
.\tools\Собрать_вопросы.ps1
.\tools\Собрать_фронты.ps1
.\tools\Собрать_индекс_персонажей.ps1
.\tools\Собрать_индекс_локаций.ps1
.\tools\Проверить_инструменты.ps1
.\tools\Завершить_ход.ps1
```

После любых ручных правок по кампании запускай `.\tools\Завершить_ход.ps1`. Если менялся только один узкий слой, можно запустить профильный скрипт, но финальная проверка все равно должна пройти через `.\tools\Проверить_проект.ps1`.
